# Career Simulator — ゲームデザインドキュメント

## 目的・ビジョン

「いろんな職業を体験しながら、自分だけのキャリアを築く」シミュレーション。
プレイヤーはフリーターからスタートし、スキル習得・転職・投資を通じて
社会的地位と経済力を築いていく。

**ターゲット体験**: 「プログラミング講座を受けてプログラマーになるか、
ビジネスセミナーに通って営業に進むか…どっちが先に稼げるようになる？」
という**キャリア戦略の判断**を楽しむ。

---

## コアコンセプト: 「選択と成長」

**コア体験**: 限られたリソース（時間とお金）をどのスキルに投資するか判断し、
正しい選択でキャリアが加速する実感を得る。

**プレイヤーの感情**:
「独学で知識を上げて事務員になったけど、ここからプログラマーを目指すか
営業を目指すかで必要な投資が全然違う…技術系に行こう！」

**主軸**: 複数のキャリアパスから自分の戦略を選び、資源配分を最適化する。

## ゲームの全体構造

### 3つの柱

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  スキル投資  │    │  キャリア    │    │  資産形成    │
│  (研修/独学) │───>│  (転職)     │───>│  (投資)     │
│             │    │             │    │             │
│ 技術/営業/  │    │ 10種の職業  │    │ 貯金/株式/  │
│ 管理/知識   │    │ 給料UP      │    │ 不動産      │
└─────────────┘    └─────────────┘    └─────────────┘
      ↑                   │                   │
      └───────────────────┴───────────────────┘
                   お金の循環
```

1. **スキル投資**: お金を使ってスキルを伸ばす (即効性)
2. **キャリア (転職)**: スキルが条件を満たせばより高給な職業へ
3. **資産形成**: 余剰資金を投資してパッシブ収入を生む

### プレイの流れ

```
序盤 (0-2分)     中盤 (2-5分)      終盤 (5-10分)
─────────────    ─────────────    ─────────────
フリーター       プログラマー     マネージャー
  ↓ 独学×5        or 営業          or コンサル
事務員           ↓ 研修で分岐      ↓ 投資で加速
  ↓ 講座          ↓                ↓
Tier2職業へ      Tier3職業へ      部長→起業家
```

---

## デザイン原則

1. **情報の透明性**: 各職業の条件・給料・現在のスキルとの差分を常に見せる。
   プレイヤーが「計算して判断する」楽しさを保証する。

2. **分岐のある成長**: 技術系・営業系・管理系のパスが存在し、
   「どこに投資するか」に正解がない（状況次第で最適解が変わる）。

3. **投資と回収のジレンマ**: 研修にお金を使えばスキルが上がるが手持ちが減る。
   投資に回せばパッシブ収入が増えるが転職が遅れる。
   常に「今何に使うか」の判断が面白い状態を維持する。

4. **自動進行 + 能動的判断**: 給料は自動で入る（放置可能）が、
   研修・転職・投資はプレイヤーの能動的な判断で行う。

5. **段階的な開放感**: 新しい職業に就けるようになった瞬間の達成感を大切にする。

### Tier 1 — 入門 (条件なし〜低)

|------|-----------|------|-------------------|
| フリーター | ¥8 | なし | — |
| 事務員 | ¥20 | 知識≥5 | 知識 +0.005/tick |

### Tier 2 — 専門職 (1つのスキル特化)

| 職業 | 給料/tick | 条件 | 仕事で伸びるスキル |
|------|-----------|------|-------------------|
| プログラマー | ¥50 | 技術≥15 | 技術 +0.01/tick |
| デザイナー | ¥45 | 技術≥10, 営業≥8 | 技術/営業 各+0.005/tick |
| 営業 | ¥55 | 営業≥15 | 営業 +0.01/tick |
| 経理 | ¥40 | 知識≥15, 管理≥5 | 知識/管理 各+0.005/tick |

### Tier 3 — 管理職・専門職上位

| 職業 | 給料/tick | 条件 | 仕事で伸びるスキル |
|------|-----------|------|-------------------|
| マネージャー | ¥100 | 管理≥20, 営業≥10 | 管理 +0.01/tick |
| コンサルタント | ¥130 | 営業≥25, 知識≥20 | 営業/知識 各+0.005/tick |

### Tier 4 — 上級職

| 職業 | 給料/tick | 条件 | 仕事で伸びるスキル |
|------|-----------|------|-------------------|
| 部長 | ¥220 | 管理≥35, 営業≥20, 評判≥30 | 管理/営業 各+0.005/tick |
| 起業家 | ¥400 | 全スキル≥30, 評判≥50, 所持金≥10万 | 全スキル +0.003/tick |

### 設計意図

- **Tier 2 は分岐**: プログラマー(技術) vs 営業(営業) vs 経理(知識+管理) で投資先が異なる
- **Tier 3 は複合スキル要求**: 1つのスキルだけでは到達できず、戦略の幅が広がる
- **Tier 4 は評判ゲート**: 時間経過(仕事による評判蓄積)も必要で、一気にスキップできない
- **起業家は総合力**: 全スキル+評判+資金のすべてが必要な最終目標

## スキルシステム

### 4つのスキル (0〜100)

| スキル | 用途 | 主な習得手段 |
|--------|------|-------------|
| 技術力 | IT系職業の条件 | プログラミング講座 (+3) |
| 営業力 | 営業/管理系の条件 | ビジネスセミナー (+3) |
| 管理力 | マネジメント系の条件 | マネジメント研修 (+3) |
| 知識 | 事務/経理/コンサルの条件 | 独学 (+1, 無料) / 資格取得 (+4) |

### 評判 (0〜100)

- 全職業で毎tick +0.002 ずつ自然増加 (約8分で100)
- 資格取得で +5 のボーナス
- 部長 (≥30) と起業家 (≥50) の条件として機能
- **設計意図**: 純粋な時間ゲートとして、最上位職への到達速度を制御する

### 研修メニュー

| 研修 | コスト | 効果 | 戦略的役割 |
|------|--------|------|-----------|
| プログラミング講座 | ¥3,000 | 技術+3 | 技術系パスの主要投資先 |
| ビジネスセミナー | ¥3,000 | 営業+3 | 営業系パスの主要投資先 |
| マネジメント研修 | ¥5,000 | 管理+3 | 管理系パス、Tier3以降に必要 |
| 独学する | 無料 | 知識+1 | 序盤の無料スキルアップ |
| 資格を取る | ¥8,000 | 知識+4, 評判+5 | 知識系+評判ブースト |

## 投資システム

### 3つの投資先

| 投資先 | 1回の投資額 | 利回り/tick | 戦略的役割 |
|--------|-----------|------------|-----------|
| 貯金 | ¥1,000 | 0.002% | 低リスク、序盤の小さな足し |
| 株式投資 | ¥5,000 | 0.01% | 中盤の収入補助 |
| 不動産 | ¥50,000 | 0.03% | 終盤の大きなパッシブ収入 |

### 投資の期待リターン例

| 投資額 | 貯金 | 株式 | 不動産 |
|--------|------|------|--------|
| ¥10,000 | 0.2/tick (2/sec) | 1/tick (10/sec) | — |
| ¥50,000 | 1/tick (10/sec) | 5/tick (50/sec) | 15/tick (150/sec) |
| ¥100,000 | 2/tick (20/sec) | 10/tick (100/sec) | 30/tick (300/sec) |

### 設計意図

- **序盤**: 投資より研修に使った方が効率的（スキル→転職→給料UPの方が速い）
- **中盤**: 余剰資金を株式に回すと収入が安定する
- **終盤**: 不動産で大きなパッシブ収入を確保し、起業家の資金条件(¥10万)にも近づく

## 数値設計の根拠

### ペーシング目標

| フェーズ | 目標時間 | 到達職業 | 累計収入の目安 |
|----------|----------|----------|---------------|
| 序盤 | 0-2分 | 事務員 → Tier2 | ~¥15,000 |
| 中盤 | 2-5分 | Tier2 → Tier3 | ~¥100,000 |
| 終盤 | 5-10分 | Tier3 → 起業家 | ~¥500,000+ |

### 具体的なフロー例 (技術系ルート)

1. **開始**: フリーター (¥8/tick = ¥80/sec)
2. **~10秒**: 独学×5 (無料) → 知識5
3. **~15秒**: 事務員に転職 (¥20/tick = ¥200/sec)
4. **~30秒**: ¥3,000貯まる → プログラミング講座×1 (技術3)
5. **~90秒**: 講座×5回完了 (技術15) → プログラマーに転職
6. **~3分**: プログラマー (¥50/tick) で研修資金を稼ぐ
7. **~5分**: マネジメント研修でスキル複合 → マネージャー or コンサル
8. **~8分**: 評判が30超え → 部長への道が開く
9. **~10分**: 全スキル30+評判50+資金10万 → 起業家

## 現状の課題と強化の方向性

### 現状の課題

- **戦略の深さ**: 最適パスが1-2本に収束しやすい（技術系一択になりがち）
- **中盤の停滞感**: Tier2→Tier3の間に「作業感」が出やすい
- **投資の存在感**: 序盤は投資する余裕がなく、システムに触れるのが遅い
- **リプレイ性**: 1回クリアしたら新しい発見が少ない
- **イベント性**: 淡々とスキルを上げるだけで、ドラマがない

### 強化の方向性

#### Phase 1: 戦略の深化
- **職業ボーナス**: 各職業に固有のパッシブ効果を追加
  - 例: プログラマー経験者は研修効率+20%、営業経験者は評判獲得+50%
- **転職回数ボーナス**: 多くの職業を経験すると「汎用スキル」が上がる
- **給料交渉**: 同じ職業でもスキルが高いと給料倍率UP

#### Phase 2: イベントとダイナミクス
- **ランダムイベント**: 「ヘッドハンティング」「不景気」「バブル」など
- **ミニクエスト**: 「技術20に到達 → 特別報酬」のようなマイルストーン
- **市場変動**: 職業ごとの需要が変動し、給料にボーナス/ペナルティ

#### Phase 3: 社会への影響
- **会社経営**: 起業家になったら従業員を雇って会社を大きくする
- **業界への影響**: プレイヤーの選択で業界の景気が変わる
- **レガシーシステム**: 引退→次世代に引き継ぎ（プレステージ的な周回要素）

#### Phase 4: マルチキャリアパス
- **専門職の追加**: 医師、弁護士、研究者、アーティストなど
- **副業システム**: メイン職業+副業で複合収入
- **資格ツリー**: 資格同士の前提条件で深い育成ツリーを形成

## 職業拡張ロードマップ — 現実に近いキャリア体系

### 設計思想

現実の労働市場を「業界×職位レベル」でモデル化する。
プレイヤーは**業界を横断して転職**でき、前職の経験が次に活きるリアルな
キャリア形成を体験する。

### スキル体系の拡張

現在の4スキル+評判を、より現実的な多軸システムに拡張する。

| スキル | 説明 | 関連業界 |
|--------|------|---------|
| 技術力 | プログラミング、工学、設計 | IT、製造、建設 |
| 営業力 | コミュニケーション、交渉、人脈 | 営業、サービス、政治 |
| 管理力 | 組織運営、計画立案、リーダーシップ | 全業界の管理職 |
| 知識 | 学問的知識、論理的思考、分析力 | 医療、法律、研究、教育 |
| 創造力 | デザイン、発想、芸術的感性 | クリエイティブ、メディア |
| 体力 | 肉体的な強さ、持久力 | 建設、農業、スポーツ、飲食 |
| 評判 | 社会的信用、知名度 | 全業界の上位職、政治 |

### 「前職経験」システム

転職時に前の職業での経験が新しい職業に活きる仕組み。
これにより「回り道」が無駄にならず、多様なキャリアパスに価値が生まれる。

```
例: 営業 → プログラマーに転職した場合
  → 営業経験により「顧客折衝ができるエンジニア」としてボーナス
  → 営業力スキルの一定割合が給料ボーナスに反映

例: 調理師 → ライターに転職した場合
  → 料理知識を活かしたフードライターとして評判ボーナス
```

**実装案**: 職種ごとに「経験済み」フラグを持ち、特定の組み合わせで
シナジーボーナスが発生する。

### 年齢・ライフイベントシステム (将来構想)

リアリティを追求するなら、年齢とライフイベントの概念を導入する。

- **年齢**: ゲーム日数に連動、年齢によって転職のしやすさが変化
- **結婚/子供**: 支出が増えるが幸福度ボーナスでスキル効率UP
- **住宅購入**: 大きな支出だが安定感ボーナス
- **退職金**: 長く同じ職業にいると退職時にボーナス
- **年金**: 高齢になると年金収入が発生

## 技術仕様

### ファイル構成

```
src/games/career/
├── DESIGN.md   ← このファイル
├── mod.rs      Game trait 実装、入力ディスパッチ
├── state.rs    データ構造 (CareerState, JobKind, etc.)
├── logic.rs    純粋関数 (tick, buy_training, apply_job, invest)
└── render.rs   読み取り専用レンダリング (3画面)
```

### Pure Logic Pattern 準拠

- `state.rs`: データのみ、ロジックなし
- `logic.rs`: `&mut CareerState` を受け取る純粋関数、IO なし
- `render.rs`: `&CareerState` を読み取り専用で描画

### テストカバレッジ

- state: 初期状態、日数計算、ログ、データ妥当性
- logic: 給料計算、スキル成長、研修購入、転職条件、投資、数値フォーマット
- mod: 入力→画面遷移→状態変化の統合テスト
