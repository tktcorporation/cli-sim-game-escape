# Dungeon Dive — コアコンセプト: 「もう一部屋だけ…の誘惑と恐怖」

## コア体験

プレイヤーが「もう少し奥に行けばもっといいものが手に入る…」と思いながらも
「でもここで死んだら全部失う…」と葛藤する、この緊張感がゲームの核心。

## プレイヤーの感情

「あと1部屋でボーナス+60G…でもHP残り30%…薬草も残り1個…行くか？帰るか？」

## 主軸

**リスク・リワードの判断** — 奥に進むほど報酬は増えるが、死のリスクも上がる。
生きて帰れば帰還ボーナスが得られるが、欲張って死ねば獲得した金を失う。

## デザイン原則

- **帰還ボーナス**: 生きて町に帰ると、探索した部屋数×階層に応じたボーナス金が得られる
- **リスクの可視化**: HP残量、残りアイテム、帰還ボーナス額を常に表示
- **戦闘の読み合い**: 敵の特殊行動予告で「今シールドを使うべきか」の判断が生まれる
- **属性弱点**: スキルの使い分けで「どのスキルを使うか」の戦略が生まれる
- **演出で感情を動かす**: クリティカルヒット、チャージ攻撃、弱点表示でフィードバックを強化

## 改善項目

### Phase 1: コアループ強化
1. **帰還ボーナス** — 部屋クリア数×階層でゴールドボーナス。死亡時は獲得金全額ロス
2. **クリティカルヒット** — 10%で1.5倍ダメージ。「会心の一撃！」表示
3. **敵のチャージ攻撃** — 一部の敵が「力を溜めている！」→次ターン2倍ダメージ

### Phase 2: 戦闘深化
4. **新スキル4種** — アイスブレード(Lv3)、サンダー(Lv5)、ドレイン(Lv6)、バーサク(Lv8)
5. **属性弱点** — Fire/Ice/Thunder。弱点スキルで1.5倍ダメージ

### Phase 3: 演出強化
6. **バトルログ改善** — ダメージ量に応じた表現バリエーション
7. **雰囲気テキスト** — HP低下時の緊迫表現、階層テーマ性

# 探索リデザイン: Overworld 化

## 方針

「最初に語り → 受付メニュー → ダンジョン」の3段構造をやめ、
**起動直後からグリッドマップ上を自由に歩ける**構造に統一する。
町＝リストメニュー、ダンジョン＝マップ探索、という非対称をなくす。

会話・買い物・依頼受託・宿泊・祈祷など、これまで Town のリスト選択肢だった
ものはすべて **地上マップ (Overworld) のタイル** として配置し、プレイヤーが
歩いて辿り着いてモーダル（既存の `DungeonEvent` ポップアップ）を開く形に
する。

## シーン構成

旧:
- `Scene::Intro(0)` / `Scene::Intro(1)` — 受付嬢の語り＋初期装備配布
- `Scene::Town` — テキスト＋選択肢リスト
- `Scene::DungeonExplore` — グリッド探索
- `Scene::GameClear`

新:
- `Scene::Overworld` — 村の **固定** グリッドマップ。敵は出現しない
- `Scene::DungeonExplore` — 据え置き
- `Scene::GameClear` — 据え置き
- Intro / Town は削除

ゲーム起動 → いきなり `Overworld` のプレイヤーキャラが村の入り口に立っている。

## 村マップ (Overworld)

**固定レイアウト**を採用。毎回同じ場所に同じ建物がある安心感を優先する。
プレイヤーが「武具屋は北東」「ダンジョンは北の奥」と空間記憶できることが、
RPG的な"ホームタウン"の没入感に直結する。

`dungeon_map.rs` の3×3セクション生成器は使わず、ハードコードした
タイル配列で `DungeonMap` を組み立てるヘルパー (`overworld_map.rs`) を
新設する。生成器のインタフェース (`DungeonMap` を返す) は流用するので、
描画・移動・タップ処理はそのまま動く。

### 配置するタイル種

既存の `CellType` を拡張:

- `DungeonEntrance` — 踏むと `enter_dungeon(state, 1)` で B1F へ
- `Shop` — 武具屋。踏むと `Overlay::Shop` を開く
- `QuestBoard` — 掲示板。踏むと `Overlay::QuestBoard` を開く
- `Inn` — 宿屋。踏むと「10Gで休む？」のイベントモーダル
- `Shrine` — 祭壇。踏むと `Overlay::PrayMenu` を開く（既存 `Idol` を村専用に流用も可）
- `VillagerNpc` — 村人NPC。踏むと会話モーダル（既存 `Npc` 拡張）

### 村の固定レイアウト案 (15×11 程度)

```
##############################
#......##............##......#
#.受付.##....広場....##.武具.#
#......##............##.屋...#
###..######........######..###
....................##.......#
#####...掲示板...#####.....###
#...#............#...#......#
#.宿#............#.祭#.入口.#
#...#............#...#..↓..##
#####.....🚶.....##############
##############################
```

- プレイヤーは下中央にスポーン
- 「広場」中央に村人NPCが数人歩いている（チュートリアル分散用）
- ダンジョン入口は右下の階段タイル

## 初期チュートリアルの分散

旧 `Intro` で一括配布していたものを、村人NPCとの会話イベントに分解:

- **受付嬢NPC**: ダンジョンの説明、初回限定で「薬草x3 / パンx2 / 50G」を渡す
- **武具屋のおっちゃん**: 初回限定で「木の剣 / 旅人の服」を渡す（以降はショップとして機能）
- **広場の村人**: 「奥に魔王がいるらしい…」等のフレーバー

これにより「歩いて世界に触れて準備が整う」体験になり、テキストを読まされる
だけのIntro画面が消える。

## 旧Townからの近道（QoL）

歩く距離が増えるトレードオフを緩和するため、頻用機能には近道を残す:

- **持ち物 / ステータス / スキル** — 既存どおり B ボタン (overlay)
- **宿屋・ショップ・掲示板** — マップ歩行のみ。近道は付けない（村に来た意味を残す）
- 死亡 / 帰還の巻物使用時は村のスポーン地点に戻る

## 既存コードへの影響

削除:
- `Scene::Intro(_)` と `advance_intro` (`logic.rs:107`)
- `Scene::Town`、`town_choices`、`update_town_text`、`execute_town_choice`
- 関連する `handle_intro_*` / `handle_town_*` ハンドラ (`mod.rs:139-195`)
- `render_intro` / `render_town_content` のリスト描画

追加:
- `Scene::Overworld`
- `CellType::{DungeonEntrance, Shop, QuestBoard, Inn, Shrine, VillagerNpc}`
  （`Shrine` は既存 `Idol` 流用可）
- `src/games/rpg/overworld_map.rs` — 固定マップ生成 (`DungeonMap` を返す)
- `events.rs` に各タイルのイベント生成（既存 `npc_event` / `peddler_event` の延長）
- `actions.rs` に新タイル踏み込み時のアクション解決（`OpenShop`, `OpenQuestBoard`,
  `RestAtInn`, `EnterDungeon` など）

更新:
- 死亡 / `ReturnToTown` / `ReturnScroll` の戻り先を `Scene::Town` →
  `Scene::Overworld`（村スポーン地点にプレイヤー再配置）
- 既存テスト約10〜15本（`town_*`, `intro_*`）を Overworld 用に書き直し

## 実装フェーズ

1. **F1: マップ基盤** — `Scene::Overworld`、`overworld_map.rs` 固定マップ、
   起動時に `Overworld` で開始するよう `RpgState::new()` を変更
2. **F2: タイル種追加** — `CellType` 拡張、踏み込み時のアクション解決
3. **F3: NPC会話と初期装備配布** — 受付嬢・武具屋NPCで Intro 機能を再現
4. **F4: 旧シーン削除** — Intro / Town 関連コードとテストを除去
5. **F5: 戻り先の差し替え** — 死亡・帰還時の遷移先を Overworld に
6. **F6: フレーバー強化** — 広場村人、看板、池などの装飾タイル
