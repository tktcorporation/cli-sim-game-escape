# 廃墟カフェ復興記 — シナリオバイブル

## ファイル構成

| ファイル | 内容 | 主な読者 |
|---------|------|---------|
| [WORLD.md](./WORLD.md) | 世界観・舞台設定・空間の物語性 | シナリオ執筆時に空気感を確認 |
| [CHARACTERS.md](./CHARACTERS.md) | キャラ設定（3層構造）・接客→会話分岐テーブル・感情曲線 | キャラのセリフ・行動に迷った時 |
| [STYLE_GUIDE.md](./STYLE_GUIDE.md) | 文体ルール・ノベルADV演出技法・ターミナルUI演出 | 書き始める前に必ず確認 |
| [PLOT.md](./PLOT.md) | プロット構成・伏線管理・感情設計・運用設計 | チャプター執筆・新コンテンツ追加時 |
| [../DESIGN.md](../DESIGN.md) | ゲームシステム設計・ソシャゲ運用基盤 | 実装・メカニクスの確認 |

## 使い方

### シナリオを書くとき

1. **STYLE_GUIDE.md** を読んで文体ルール＆ノベルADV演出技法を確認
2. **CHARACTERS.md** で該当キャラの口調・内面・「今の感情状態」を確認
3. **PLOT.md** でそのチャプターの感情設計・伏線・「引き」を確認
4. **WORLD.md** で舞台の空気感＆空間の変化を確認

### 特別接客シーンを書くとき

1. **CHARACTERS.md** の「接客→会話分岐テーブル」でメニューと反応を確認
2. **STYLE_GUIDE.md** の「VA-11 Hall-A式」セクションで設計意図を再確認
3. 「正解を当てるゲーム」ではなく「この人を理解しているか」の体験を意識

### 新キャラを追加するとき

1. **CHARACTERS.md** の設計チェックリストに沿って設定（3層構造必須）
2. 接客→会話分岐テーブルを作成（好み/挑戦/外しの3パターン以上）
3. 感情曲線をチャプターマップに追加
4. **PLOT.md** のチャプター構成に登場箇所を追加
5. **WORLD.md** に関連する場所があれば追加

### 新チャプター/サイドクエストを追加するとき

1. **PLOT.md** の感情設計表を確認し、感情の流れに矛盾がないか確認
2. 伏線管理表を更新（仕込み・回収・「3回触れてから回収」の原則）
3. 解放条件をDESIGN.mdのシステムと整合させる
4. サイドクエストはテンプレートに沿って構成

### デイリー会話を追加するとき

1. **PLOT.md** のデイリー会話バリエーション管理で、親密度帯を確認
2. **CHARACTERS.md** の口調サンプルでテイストを合わせる
3. **STYLE_GUIDE.md** の感情表現ストックを活用
