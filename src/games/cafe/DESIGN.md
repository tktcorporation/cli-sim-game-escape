# 廃墟カフェ復興記 — ゲームデザインドキュメント v1

## 目的・ビジョン

廃墟になったカフェを引き継いだ主人公が、レシピを発見し、常連客と絆を深めながら
店を復興させる**ソシャゲ型シナリオ経営SLG**。

一本道のメインストーリーを軸に、経営を進めることでシナリオが解放される
イベント駆動型の構造。アークナイツ/ブルーアーカイブのような**リアル時間ベースの
ソシャゲ運用モデル**を採用し、デイリーミッション・ログインボーナス・スタミナで
継続プレイを促す。

シナリオはラノベ/学園アイドルマスター的な「読み物」として構成し、
キャラクターの内面に寄り添う濃密な描写を重視する。

**ターゲット体験**:
「今日のログボでレシピヒントが手に入った！新しい組み合わせで
抹茶ラテを開発して…常連の佐倉さんが喜んでくれた。親密度MAXで
解放されたエピソード、泣ける…。明日のデイリーも回さなきゃ」

## コアコンセプト: 「発見と繋がり」

### 3つの柱

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  1. レシピ発見   │  │  2. ストーリー   │  │  3. ソシャゲ運用 │
│  (経営の深さ)    │  │  (感情の深さ)    │  │  (継続の仕組み)  │
│                  │  │                  │  │                  │
│ 素材の組み合わせ │  │ 一本道メイン     │  │ デイリー/ログボ/ │
│ で新メニュー発見 │  │ +キャラエピソード │  │ スタミナで毎日   │
└────────┬─────────┘  └────────┬─────────┘  └────────┬─────────┘
         │                     │                     │
         └──────── 経営→解放→継続 ────────┘─────────────────┘
```

1. **レシピ発見** — 素材の組み合わせで新メニューを発見。客層・売上が変化する戦略性
2. **ストーリー** — 条件達成でメインストーリー＆キャラエピソードが解放。読み物として楽しめる
3. **ソシャゲ運用** — リアル時間ベースのデイリー/ログボ/スタミナで毎日起動する動機を作る

### プレイヤーの感情

**レシピ発見**: 「コーヒー豆×ミルク×抹茶で…抹茶ラテ！？
これ原価安いのに売値高い、メニューに入れよう」

**ストーリー**: 「佐倉さん、元パティシエだったのか…自分の店を
失った過去がこのカフェと重なって…続き早く読みたい」

**ソシャゲ運用**: 「ウィークリー全達成でサイドクエスト解放される！
あと営業2回でコンプ…スタミナ回復待とう」

### 勝利条件（ストーリークリア）

| 条件 | 名前 | 説明 |
|------|------|------|
| **メインクリア** | 復興完了 | メインストーリー最終章クリア |
| **キャラコンプ** | 絆の証 | 全常連客の親密度MAX＆エピソード全解放 |
| **レシピコンプ** | 伝説のメニュー | 全レシピ発見＆隠しレシピ解放 |

## デザイン原則

1. **シナリオファースト**: ゲームメカニクスはシナリオを届けるための手段。
   「次のストーリーを読みたい」がプレイの最大動機になるよう設計する。

2. **発見の快感**: レシピは教えない。素材の組み合わせをヒントから推理し、
   自分で発見する体験を大切にする。

3. **キャラクターの呼吸**: 常連客は単なるパラメータではない。
   通うたびに会話が変わり、関係が深まり、やがて本音を見せる。

4. **日常の心地よさ**: 毎日カフェを開ける「ルーティン」が心地よい。
   デイリーは義務ではなく、カフェの日常を楽しむ行為。

5. **読み物としての完成度**: シナリオはラノベ品質。
   スキップしたくならない、読むこと自体が報酬になるテキスト。

## システム1: レシピ発見（ガチャに代わるコア発見メカニクス）

### 概要

素材同士を組み合わせることで新しいメニュー（レシピ）を発見する。
「ガチャ」の代わりに「発見」が報酬の核になる。

### 素材カテゴリ

| カテゴリ | 初期素材例 | 解放条件 |
|---------|-----------|---------|
| ベース | コーヒー豆、紅茶葉、ほうじ茶 | 初期 |
| 乳製品 | 牛乳、生クリーム | Ch.1クリア |
| フルーツ | レモン、イチゴ、リンゴ | Ch.2クリア |
| 甘味 | 砂糖、蜂蜜、黒糖 | 初期 |
| スパイス | シナモン、バニラ、抹茶 | Ch.3クリア |
| プレミアム | カカオ、和三盆、季節限定素材 | 特殊条件 |

### 組み合わせルール

- レシピ = ベース素材1 + サブ素材0〜2
- 組み合わせ結果は事前定義テーブルで管理（ランダムではない）
- 未発見レシピにはヒントが出る（「苦いベースに甘い乳製品を…」）
- 素材5種→組み合わせ20+、素材10種→100+のスケーラビリティ

### レシピ属性

```
struct Recipe {
    name: String,
    ingredients: Vec<Ingredient>,
    cost: u32,         // 原価
    price: u32,        // 売値
    appeal: HashMap<CustomerTag, i32>,  // 客層ごとの魅力度
    unlock_condition: Option<UnlockCondition>,
}
```

## システム2: 営業サイクル（ハイブリッド型）

### 1日の流れ

| フェーズ | 操作 | 内容 |
|---------|------|------|
| **朝・仕入れ** | 手動 | スタミナ消費で素材仕入れ。メニュー構成を決定 |
| **昼・営業** | 自動+手動 | 基本は自動ログ。「！」付き特別客のみ手動接客 |
| **夜・振り返り** | 閲覧 | 売上レポート、常連客来店ログ、イベント判定 |

### 営業フェーズ詳細（ハイブリッド型）

- **通常客**: 自動でテキストログが流れる（「OLが来店→カフェラテを注文→満足！」）
- **特別客（！マーク）**: 手動接客。メニューを選んで提案する
  - 常連客: 好みに合うメニューで親密度UP
  - ストーリー客: 特定条件でイベントトリガー
- 1日の営業で通常客5〜10人、特別客1〜3人
- 特別客への対応がストーリー進行に直結

## システム3: 常連客＆親密度

### 概要

常連客は5〜8人の固定キャラ。ガチャではなくストーリー進行で順次登場。
通い続けると親密度が上がり、キャラエピソードが解放される。

### 親密度ランク

| ランク | 必要親密度 | 解放要素 |
|-------|-----------|---------|
| ★☆☆☆☆ | 0 | 初来店イベント、基本会話 |
| ★★☆☆☆ | 100 | 名前を覚えるイベント、好みが判明 |
| ★★★☆☆ | 300 | サイドエピソード前編、裏メニューリクエスト |
| ★★★★☆ | 600 | サイドエピソード後編、特殊スキル解放 |
| ★★★★★ | 1000 | 最終エピソード、エンディング分岐に影響 |

### 親密度の上げ方

- 好みに合うメニューを提供: +10〜30/回
- 特別接客で正解: +50
- イベント選択肢で好感度UP選択: +30〜100
- デイリーミッション達成で全体+5

## システム4: ソシャゲ運用基盤

### 4-A: スタミナ（仕入れ予算）

世界観に溶け込む名称で理性/APと同じ機能を実現。

| パラメータ | 値 |
|-----------|-----|
| 最大値 | 100 |
| 回復速度 | 6分に1pt（=10pt/時、満タンまで10時間） |
| 営業1回コスト | 20〜40（仕入れ内容による） |
| 1日あたり | 3〜5回営業可能 |
| 溢れ防止 | 上限超過分は消滅（毎日ログインの動機） |

### 4-B: デイリーミッション

毎日04:00（JST）リセット。

| ミッション例 | 報酬 |
|-------------|------|
| 営業を1回行う | 資金×100 |
| 特別客を1人接客する | 素材チケット×1 |
| 新しい組み合わせを試す | レシピヒント×1 |
| 常連客に好みのメニューを出す | 親密度+10 |
| **全達成ボーナス** | **ストーリーキー×1** |

**ウィークリーミッション**: 7日間で達成。全達成報酬はサイドクエスト即解放。

### 4-C: ログインボーナス

月間カレンダー型、累計日数カウント（途切れてもリセットしない＝ブルアカ方式）。

| 日数 | 報酬例 |
|------|--------|
| 1日目 | 資金×200 |
| 3日目 | 素材チケット×3 |
| 7日目 | レアレシピヒント |
| 14日目 | 限定素材 |
| 21日目 | ストーリーキー×3 |
| 28日目 | 限定レシピヒント（月替わり） |

3日以上離脱後の復帰ボーナスあり（離脱ペナルティなし＝復帰しやすい設計）。

## システム5: ストーリー構造

### メインストーリー

- 一本道。チャプター制（Ch.0〜Ch.8+）
- 解放条件: 累計売上、レシピ数、常連客数などの経営マイルストーン
- 「カフェが廃墟になった理由」を軸に、前オーナーの謎を追うミステリー要素
- 各チャプター末に「引き」（クリフハンガー）を設計

### サイドクエスト

- 常連客ごとの個別エピソード（親密度で解放）
- 季節イベント（月替わりコンテンツ）
- 隠しレシピに紐づくミニストーリー

### シナリオ解放フロー

```
経営プレイ → マイルストーン達成 → メインストーリー解放
    ↓
常連客接客 → 親密度UP → キャラエピソード解放
    ↓
デイリー/ウィークリー → ストーリーキー → サイドクエスト解放
```
