# 廃墟カフェ復興記 — ゲームデザインドキュメント v1

## コア体験

廃墟になったカフェを引き継いだ主人公が、レシピを発見し、常連客と出会い、
店の過去の謎を解き明かしながら復興させていく。
**シナリオを読み進める喜び**が主軸で、カフェ経営はストーリーを進めるエンジン。

## プレイヤーの感情

「売上が目標に届いた…次のチャプターが解放された！前オーナーが店を畳んだ理由って…
あの常連客が何か知ってるのか？早く続きが読みたい、もう少し経営を回そう」

## 主軸

**シナリオ駆動の経営SLG** — 一本道のメインストーリーを章立てで追加運用できる構造。
経営の成果がストーリー解放のトリガーとなり、ストーリーが経営の動機になる循環。

---

## デザイン原則

1. **シナリオが引力、メカニクスが滞在時間**
   ストーリーの続きが気になるからカフェを回す。カフェが面白いから待ち時間も苦にならない。

2. **一本道+サイドクエストで運用コスト最小化**
   メインストーリーは分岐なし。章を末尾に追加するだけで運用可能。
   サイドクエストは常連客ごとの独立エピソードで、順不同に追加できる。

3. **発見の喜び**
   レシピは素材の組み合わせで「発見」する。正解を教えず、試行錯誤と偶然の出会いを演出。
   新レシピ → 新しい客層 → 新しいストーリー の連鎖。

4. **情報は見せる、答えは見せない**
   素材の特徴（甘い・苦い・香りが強い）はヒントとして見せる。
   だが最適な組み合わせは試さないとわからない。

5. **コンテンツ飽和の回避**
   レシピ発見の組み合わせ爆発で、少ない素材定義から多くのコンテンツを生む。
   ストーリーは追加型なので、既存コンテンツが陳腐化しない。

## システム1: カフェ経営

### 1日の流れ

| フェーズ | 操作 | 内容 |
|----------|------|------|
| 朝・仕入れ | 能動 | 素材を選んで購入。予算内でやりくり |
| 朝・メニュー | 能動 | 発見済みレシピから今日のメニューを3品選択 |
| 昼・営業 | 自動 | 客が来店→注文→提供。ログが流れる |
| 夜・結果 | 閲覧 | 売上・評判・客数の集計。条件チェック |

### 売上の仕組み

```
売上 = Σ (メニュー価格 × 注文数)
注文数 ∝ 評判 × メニューと客層のマッチ度
評判 = f(累計満足度, 連続営業日数)
```

## システム2: レシピ発見

### 素材

素材には**特性タグ**がある:

| 素材 | 特性 | 入手 |
|------|------|------|
| コーヒー豆 | 苦味, 香り | 初期 |
| 牛乳 | まろやか | 初期 |
| 砂糖 | 甘味 | 初期 |
| チョコレート | 甘味, 苦味 | Ch.1クリア |
| 抹茶 | 苦味, 和風 | Ch.2クリア |
| バニラ | 香り, 甘味 | サイドクエスト |
| 果物 | 酸味, 彩り | 季節イベント |
| スパイス | 辛味, 香り | Ch.3クリア |
| ... | ... | ... |

### レシピ発見メカニクス

- 素材を2〜3個選んで「試作」する
- 特性の組み合わせでレシピが決定される
- 成功 → レシピ登録、失敗 → 「微妙な味…」（ヒントは出る）
- 一部のレシピはストーリーでヒントが語られる

```
例: コーヒー豆 + 牛乳 → カフェラテ (定番)
    コーヒー豆 + チョコレート + 牛乳 → カフェモカ (人気)
    抹茶 + 牛乳 + 砂糖 → 抹茶ラテ (和風好き客に刺さる)
```

### 客層マッチング

レシピには**タグ**が付き、客層ごとに好むタグが異なる:

| 客層 | 好むタグ | 出現条件 |
|------|----------|----------|
| サラリーマン | 苦味, 定番 | 初期 |
| 学生 | 甘味, 安い | 初期 |
| マダム | 香り, 上品 | 評判10+ |
| 観光客 | 和風, 彩り | Ch.2以降 |
| 美食家 | 複雑, 希少 | 評判30+ |

## システム3: シナリオ

### 構造

```
メインストーリー (一本道、章追加で運用)
├── Ch.0: プロローグ — カフェを引き継ぐ
├── Ch.1: 最初の常連 — 店の噂を聞きつけた客
├── Ch.2: 前オーナーの手紙 — 引き出しから見つかる手紙
├── Ch.3: ライバル店登場 — 近所に新しいカフェが…
├── Ch.4: 前オーナーの娘 — 「父の店を返して」
├── Ch.5: 真実 — なぜ店は廃墟になったのか
├── ...（追加可能）
└── 最終章: 復興完了

サイドクエスト (常連客ごと、独立)
├── 佐藤さんの秘密 — 毎日来る無口な常連の事情
├── 学生グループ — 試験前の勉強会と友情
├── 謎の評論家 — 厳しい評価の裏にある本心
└── ...（追加可能）
```

### チャプター解放条件

| チャプター | 解放条件 |
|------------|----------|
| Ch.0 | ゲーム開始 |
| Ch.1 | 累計売上 500 達成 |
| Ch.2 | レシピ 5種発見 |
| Ch.3 | 評判 15 到達 |
| Ch.4 | Ch.3クリア + 営業30日 |
| Ch.5 | 全サイドクエスト1つ以上クリア |

### サイドクエスト解放

常連客が一定回数来店すると「！」マークが出現 → 会話イベント開始。
3〜5回の会話イベントで1つのサイドクエスト完結。
報酬: 新素材、レシピヒント、設備、称号など。

## システム5: デイリーミッション

### 設計思想 (アークナイツ/ブルアカ参考)

デイリーミッションの目的は**「今日もう1回開こう」の動機付け**。
個別ミッションは軽く、全達成ボーナスで「あと1個」を誘う構造。

### デイリーミッション (リアル1日でリセット)

毎日04:00にリセット。3〜4個のミッション+全達成ボーナス。

| ミッション例 | 条件 | 報酬 |
|-------------|------|------|
| 営業を行う | 1日サイクルを1回完了 | ¥50 |
| レシピを試作する | 試作を2回行う | 素材×1 |
| 常連客に提供 | 常連が注文したメニューを出す | 親密度+1 |
| 売上目標達成 | 1日の売上が¥200以上 | ¥30 |
| **全達成ボーナス** | 上記すべて完了 | **試作チケット×1** |

### ミッションのデータ構造 (運用しやすさ重視)

```rust
// ミッションはテンプレート+パラメータで定義
// 新しいミッションタイプの追加 = enum variant追加 + 条件判定1関数
enum MissionType {
    CompleteDayCycle { count: u32 },    // 営業N回
    CraftRecipe { count: u32 },         // 試作N回
    ServeRegular,                       // 常連に提供
    DailySales { target: u32 },         // 売上目標
    DiscoverRecipe,                     // 新レシピ発見
    // ...追加容易
}
```

### ウィークリーミッション (月曜04:00リセット)

デイリーより大きな目標。1週間かけて達成する。

| ミッション例 | 条件 | 報酬 |
|-------------|------|------|
| 経営の達人 | 7日間営業する | 設備素材×1 |
| レシピ研究家 | 新レシピを2種発見 | 希少素材×1 |
| 人気店への道 | 累計売上¥2000 | 看板装飾 |
| **全達成ボーナス** | 上記すべて完了 | **ストーリーキー×1** |

**ストーリーキー**: サイドクエストを1つ即座に解放できるアイテム。
シナリオというメインの引力に直結する報酬にすることで、ウィークリーの動機を強くする。

## システム6: ログインボーナス

### 設計思想

アークナイツ/ブルアカ式の**月間カレンダー型**。
連続ログインではなく**累計ログイン日数**でカウント（途切れてもリセットしない）。
→ 途切れた時の「もう取り戻せない」絶望感を排除し、復帰しやすくする。

3日以上ログインしなかった場合、復帰時に特別ボーナス:
- 「お帰りなさい！」メッセージ + 素材パック
- 離脱を罰しない設計（ブルアカの復帰ログボと同思想）

## システム7: スタミナ（行動力）

### 設計思想

アークナイツの「理性」、ブルアカの「AP」に相当。
カフェゲームでは**「仕入れ予算」**として表現する。

### 仕組み

```
仕入れ予算: リアル時間で回復するリソース
├── 最大値: 100（設備で増加可能）
├── 回復速度: 6分に1ポイント（1日で最大240回復）
├── 1日の営業コスト: 20〜40（メニュー・素材による）
└── 1日あたり約6〜12回営業可能
```

| 項目 | 値 | 備考 |
|------|-----|------|
| 最大値 | 100 | 設備「大型冷蔵庫」で+20 |
| 回復速度 | 1pt / 6分 | 1時間で10pt |
| 通常営業コスト | 20 | 基本メニュー3品 |
| 高級営業コスト | 40 | 希少素材メニュー含む |
| 試作コスト | 10 | レシピ発見の試行 |

### なぜ「仕入れ予算」か

- 「スタミナ」だとゲーム的すぎる。「仕入れ予算」なら世界観に溶け込む
- 予算が回復する = 「卸業者との取引枠が翌日リセットされる」と解釈可能
- プレイヤーは「今の予算で高級メニューに挑戦するか、通常で回数を稼ぐか」を判断

### オーバーフロー防止

上限を超えて回復しない → 「溢れる前にログインして使おう」の動機。
ただし**ログインボーナスの予算回復は上限を超えて加算**（ブルアカ方式）。
